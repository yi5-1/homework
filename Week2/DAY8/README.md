## 怎麼開始（給每組）

1. **選題** → 進到該專題資料夾讀 `README.md`。
2. **跑起手式** → 每個專題資料夾裡都附了 `起手式/`，是一份**已經能跑的最小骨架**（含 `requirements.txt`）。先 `pip install -r 起手式/requirements.txt` 把它跑起來，確認環境沒問題。
3. **接著加功能** → 骨架裡用 `TODO` 標出該補的地方；README 也標了「從哪個 Week1 檔案改起」。**先能跑再加功能**，不要從空白檔開始。
4. **分工** → 把「必做功能」拆成幾條，一人顧一條。
5. **每天能 demo** → 寧可功能少但跑得動，也不要一堆半成品。

在自己的資料夾裡開檔開發，例如：`Week2/DAY8/專題A_智慧視覺打卡系統/我的實作/`。

## 專案慣例（跟 Week1 一致）

- **檔名、函式、變數可以用繁體中文**（例：`def 計算出勤率():`、`灰色影像 = cv2.cvtColor(...)`）——這是本課刻意的風格，方便對照。
- matplotlib / tkinter 中文顯示：`plt.rcParams['font.sans-serif'] = ['Microsoft JhengHei']`，避免方框。
- 密碼、資料庫連線放 `.env`（用 `python-dotenv` 讀），**不要 commit**；`.env` 已在根目錄 `.gitignore`。
- 資料庫檔（`*.db`）、圖片（`*.jpg`）已被 gitignore，各人跑起來自己會產生。

## 交付

在自己的專題資料夾內放：
```
我的實作/
├── README.md      # 怎麼安裝、怎麼跑、功能清單、分工
├── (你的程式碼)
└── requirements.txt   # 這個專題要 pip install 什麼
```
demo 當天能從乾淨環境 `pip install -r requirements.txt` 後跑起來即可。
