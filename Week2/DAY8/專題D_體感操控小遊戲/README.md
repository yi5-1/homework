# 專題 D — 體感操控小遊戲

不用鍵盤，用鏡頭偵測「手的位置」或「某個顏色的物體」來控制一個 pygame 小遊戲。把電腦視覺變成搖桿。

**整合**：DAY7（OpenCV 偵測）＋ DAY5（pygame 遊戲）＋ DAY2（座標 / 邏輯）
**難度** ★★★☆ ｜ **建議 2 人**

---

## 系統架構

```
鏡頭 (OpenCV)  →  偵測目標位置 (x, y)  →  換算成遊戲控制量  →  pygame 更新玩家
   每一格：找顏色色塊中心 / 手的位置          （左右移動、閃避）      畫面渲染
```

核心是：**每一個遊戲迴圈都讀一張鏡頭影像，算出一個座標，拿去控制角色**。OpenCV 和 pygame 在同一個 while 迴圈裡輪流跑。

---

## 必做功能

1. OpenCV 穩定偵測一個目標的位置：
   - 顏色法（簡單）：`cv2.inRange` 抓特定顏色（例：紅色手套 / 色卡），用 `cv2.findContours` 找最大色塊中心。
   - 或形狀法：參考 DAY7 霍夫圓 / 凸包偵測。
2. 一個能玩的 pygame 遊戲（接球 / 閃隕石 / 走迷宮，簡單就好）。
3. 把偵測到的位置換算成角色控制（例：色塊在畫面左邊 → 角色往左）。
4. 有分數 / 生命 / 結束畫面。

## 加分功能

- 手勢分兩種動作：張開 / 握拳（用色塊面積或凸包缺陷數）→ 對應「移動」和「射擊 / 跳」。
- 校準畫面：開始前讓玩家框選要追蹤的顏色，適應不同環境光。
- 用 DAY5 的多人架構做「兩人各一台鏡頭連線對戰」（挑戰級）。
- 把鏡頭小畫面疊在遊戲角落，讓玩家看到自己被偵測到的點。

## 從哪裡改起

| 你要的功能 | 抄這個檔 |
|---|---|
| 開鏡頭 + 逐格處理 | `Week1/DAY7/視覺.py` |
| 找形狀 / 輪廓中心 | `Week1/DAY7/凸包檢測(幾何形狀分類).py`、`霍夫圓檢測.py` |
| 即時調參數（找顏色門檻）| `Week1/DAY7/視角切換與形狀偵測.py`、`tkinter影像處理工具.py` |
| pygame 遊戲迴圈 / 移動 / 碰撞 | `Week1/DAY5/pygame_multiplayer/game.py`、`server.py` |
| 多人連線（挑戰）| `Week1/DAY5/pygame_multiplayer/` |

> **效能提醒**：OpenCV 讀影像不要每格做太重的運算，否則遊戲會卡。先把偵測簡化（縮小影像、只找顏色），順了再加。

## 需要安裝
```bash
pip install opencv-python numpy pygame
```

## 驗收 demo
1. 對鏡頭移動你的手 / 色卡，遊戲角色跟著動。
2. 完整玩一局到結束畫面，分數正確。
3. 說明你怎麼把「影像座標」換成「遊戲控制」。

## 整合：對接專題 B（多人網頁排行榜）

這個遊戲已經預留好呼叫**專題 B**排行榜服務的框架，兩邊各自開發、互不阻塞：

- `線上排行榜.py`：對接用的小工具，只用標準函式庫（`urllib`），呼叫 B 的
  `POST /api/score`（送分數）跟 `GET /api/leaderboard`（取前 10 名），預設打
  `http://localhost:8770`（B 的起手式預設埠號）。要接到別台機器就改這支檔案
  最上面的 `服務位址`。
- 分數在遊戲結束畫面（`ST_結束`）送出：本機仍然照存一份 `ranking.json`
  當退路；同時丟一個背景執行緒去呼叫 B 的 API。因為連不上時的 TCP 逾時可能
  拖到 1~3 秒，網路呼叫刻意放進背景執行緒，不擋畫面——結束畫面先用本機
  「今日排行」秒開，等背景執行緒把線上榜拿回來，畫面會自動換成「線上排行榜」。
- 欄位對應：B 的資料庫欄位叫 `kills`（沿用它射擊遊戲的命名），這裡直接把本
  遊戲的存活分數塞進 `kills` 送過去，B 那邊只負責排序 / 顯示數字，語意上是
  「擊殺數」還是「分數」不影響功能。

**怎麼一起測試：**
```bash
# 開一個視窗跑 B 的服務（先用起手式版本就能測）
python "Week2/DAY8/專題B_多人網頁遊戲排行榜/起手式/排行榜.py"

# 開另一個視窗跑這個遊戲
python "Week2/DAY8/專題D_體感操控小遊戲/專題_閃避隕石.py"
```
玩一局到結束畫面，應該會看到標題從「今日排行（本機）」換成「線上排行榜」，
且分數已經出現在 B 那邊 `http://localhost:8770/` 的排行榜頁面上。

> B 那邊如果之後改了路由路徑、欄位名稱，或换成正式版（併進 `web_shooter` /
> 换 PostgreSQL），只需要改 `線上排行榜.py` 裡的 `服務位址`、路徑跟欄位名稱
> 三個地方，不用動遊戲主邏輯。
